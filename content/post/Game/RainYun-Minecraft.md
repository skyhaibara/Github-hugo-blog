## 为什么别在自己电脑上开服

本地开服(单人世界"对局域网开放" 或自己跑服务端)有三个绕不过去的坑：

- **关机即停**：你电脑一关 / 一断网，朋友全掉线
- **外网连不进**：家用宽带基本是大内网 IP，朋友要连还得搞内网穿透 / 端口映射，巨折腾
- **又开服又游玩**：模组服吃内存，整合包动辄要 6~8G，自己机子一边带服一边带客户端，两头卡

丢到云服务器上，这三个问题一次性解决。

## 一、买服务器：配置怎么选

Minecraft 模组服**最吃内存**，CPU 反而是单线程为主(主频比核数更重要)。按整合包体量参考下表：

| 场景 | 玩家数 | 推荐内存 | 说明 |
|------|--------|----------|------|
| 纯净服 / 轻模组 | 2~5 人 | 4 GB | 原版或十几个模组 |
| 中型整合包 | 3~8 人 | 6~8 GB | 几十上百个模组 |
| 重型整合包(如脆骨症 NFWC2) | 3~6 人 | **8 GB 起** | 模组多、生成卡，内存给够才不崩 |

> ⚠️ 内存千万别抠。模组服内存不足会直接 `OutOfMemoryError` 崩服，这是新手翻车第一名。

## 二、两条开服路线

### 路线 A：雨云「游戏云」一键面板(新手推荐)

雨云游戏云自带网页管理面板，省掉装环境的麻烦：

1. 控制台进入你的游戏云实例 → 打开管理面板
2. 选择服务端类型 / 上传整合包服务端(见第三节)
3. 在面板里设好**内存**、**端口**、**Java 版本**
4. 点「启动」，看日志出现 `Done!` 就成了

![雨云游戏云管理面板 — 启动按钮](https://pub-fb5f542427074d66a4742ed7d0fcb694.r2.dev/rainyun-mc/panel.png)

### 路线 B：云服务器手动开(更自由)

适合想自己掌控的人，大致流程：

```bash
# 1. 装对应版本的 Java(版本要和整合包匹配,见下方提示)
sudo apt update && sudo apt install -y openjdk-21-jre-headless

# 2. 建目录、上传服务端文件到这里
mkdir -p ~/mcserver && cd ~/mcserver

# 3. 首次启动(会生成 eula.txt 然后退出)
java -Xms6G -Xmx8G -jar server.jar nogui
```

> 💡 **Java 版本对照**(选错必报错)：MC 1.16 用 Java 8、1.18~1.20.4 用 Java 17、1.20.5+ 用 Java 21。脆骨症 NFWC2 按它对应的 MC 版本来选。

## 三、上传整合包服务端

关键：整合包要用**服务端(Server Pack)**，不是你客户端那个整合包！

- 在 **CurseForge / Modrinth** 整合包页面找 **Server Pack / 服务端文件** 下载
- 解压后上传到服务器(面板直接传，或云服务器用 SFTP / `scp`)
- 里面一般带好了 `forge`/`fabric` 启动器和 `run.sh` / `start.bat`

> ⚠️ 模组服必须**客户端和服务端装一模一样的模组和加载器版本**，差一个都进不去。

## 四、关键配置

启动一次后会生成配置文件，改这几个：

- **`eula.txt`** → 把 `eula=false` 改成 `eula=true`(同意用户协议，不改不让启动)
- **`server.properties`** 常改项：

| 配置项 | 建议值 | 作用 |
|--------|--------|------|
| `server-port` | `25565` | 服务器端口(MC 默认) |
| `online-mode` | `true` / `false` | 正版验证。有人用离线号就设 `false` |
| `view-distance` | `8`~`10` | 视距，**越大越吃性能**，模组服别开太高 |
| `max-players` | 按需 | 最大人数 |
| `difficulty` | `normal` | 难度 |

- **内存参数**：在启动命令 / 面板里设 `-Xms6G -Xmx8G`(最小/最大内存，按你买的配置给)

## 五、启动 & 看日志

启动后盯一眼日志：

- 首次启动会**生成世界 + 加载所有模组**，重型整合包可能要等几分钟，别急着关
- 看到 `Done (xx.xxxs)! For help, type "help"` 就是成功了
- 中途红字 `ERROR` / `crash` 就去 `crash-reports/` 文件夹看崩溃报告

## 六、朋友怎么连

1. 客户端装上**和服务端完全一致**的整合包
2. 多人游戏 → 添加服务器 → 地址填 `你的公网IP:25565`(默认端口可省略 `:25565`)
3. 进服开玩 🎉

## 七、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 启动就 `OutOfMemoryError` | 内存给少了 | 调大 `-Xmx`，或升配置 |
| `Unsupported class version` | Java 版本不对 | 换成整合包要求的 Java 版本 |
| 朋友连不上 / 超时 | 端口没放行 | 安全组放行 25565 |
| 进服秒退 / 缺模组 | 客户端模组对不上 | 客户端装和服务端一致的整合包 |
| 跑一会儿就崩 | 某模组冲突 | 看 `crash-reports/` 定位是哪个模组 |

## 八、进阶玩法

- **网页面板管理**：云服务器可以装 [MCSManager](https://mcsmanager.com/)，网页开关服、看日志、传文件，体验接近游戏云面板
- **后台常驻**：用 `screen` / `tmux` 跑服，关掉 SSH 服也不停
- **开机自启 + 自动备份**：写个 `systemd` 服务，再加个定时 `tar` 备份存档，防止翻车丢档
